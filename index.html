01

整體策略

5 步驟
Step 1
Board Difference Matrix
釐清 reuse / modified / new
Step 2
HIL Smoke Test
boot / dmesg / device readiness
Step 3
Peripheral Integration HIL
touch / display / MCU / audio / GPS / USB / camera
Step 4
Stress / Regression HIL
cycle test / 24hr monitor / hotplug
Step 5
ASPICE Evidence Package
traceability · review · release gate
HIL MVP 原則：不要追求完整產品情境全部自動化。先做出一套能證明 SDK/BSP 修改有效的自動化測試 — 每次 build image / 修改 DTS / 修改 driver / 修改 service 之後，HIL 可以自動回答板子能否開機、kernel 有無 panic、device 是否 ready、power cycle 是否穩定。
